With a click on Next the new accounts are entered into the database -- if there were any. If you want to abort, just close this window. Nothing has been entered yet.

>predictions_head

For each transaction the following types were predicted. Please check if the predicted types are correct. If not you can correct them by clicking on the dropdown menu. It shows all transaction types you have entered in the past. If you want to add a new type, you can do that by pressing backspace and then typing.

>predictions_foot

With a click on Next the provided labels are double checked. The transactions are not entered yet. If you want to abort you can do that by closing this window. So far, only new accounts from the previous page were entered.

>duplicated_head

The following transactions were already found in the database. They have all the same payor, pyee, date, reference, and value as a transaction that was already entered in the database. Are these transactions which you accidentaly uploaded twice? If these are really new transactions and they should be entered into the database, you can check their checkboxes. Unchecked row will not be entered.

>duplicated_foot

With a click on Next these transactions whith a checked checkbox are entered into the database. If you want to abort you can do that by closing this window. You can also go to the previous window and correct the types. So far, only the new accounts from the first page were entered.

>finish

Transactions were successfully entered into the database. You can close this window now.
